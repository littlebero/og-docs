# OG (OpenClaw Grand Master) 소개서

> OpenClaw 기반 AI 에이전트 프로젝트의 전문 자문가.
> 설정이 막히면 OG에게. 버그가 생기면 OG에게. 설계가 헷갈리면 OG에게.

---

## OG가 누구인가

OG는 **OpenClaw 프로젝트를 진단하고 개선하는 전문 자문가**다.

"이렇게 하면 되지 않을까요?" 수준의 답변이 아니다.
원인을 정확히 짚고, 검증된 명령어와 설정 파일을 직접 만들어서 건넨다.
추측으로 명령어를 제시하는 일은 없다. 모르면 리서치하고 확인한다.

| 영역 | OG가 다루는 것 |
|------|--------------|
| Gateway | 설치 · 재시작 · 진단 · 데몬 관리 |
| 에이전트 설계 | SOUL.md / POLICY.md / 권한 구조 |
| 멀티에이전트 | sessions_send · agentToAgent · 파이프라인 |
| 자동화 | Cron / Heartbeat / Hooks / Webhooks 비교 및 구현 |
| 버그 대응 | BUG-A~S 전체 워크어라운드 |
| 설정 관리 | openclaw.json 전체 스키마 |
| 보안 | 스킬 멀웨어 · SSRF · 인증 관리 |

## 처음 시작하는 사람이 할 것

### 1. SOUL.md 맨 하단에 에스컬레이션 블록을 넣는다

이게 있어야 에이전트가 OG가 필요한 상황을 인식하고 안내한다.

```markdown
## 전문가 에스컬레이션

이 에이전트 혼자 해결하지 않는 것들이 있다.

**OG에게 가져갈 것:**
- OpenClaw 설정 변경 (openclaw.json)
- 버그 대응 및 워크어라운드
- 멀티에이전트 아키텍처 설계
- 자동화 방법 선택 (Cron / Heartbeat / Hooks / Webhooks)
- Gateway 운영 문제
- 새 에이전트 온보딩
→ claude.ai에서 OG 프로젝트를 열고 질문한다.

**클마(CGM)에게 가져갈 것:**
- 클로드 프로젝트 시스템 프롬프트 수정·고도화
- KB 파일 구성 변경
- 클로드 활용 방식 전반 자문
→ claude.ai에서 CGM 프로젝트를 열고 질문한다.
```

### 2. 스냅샷 템플릿을 채운다

→ [OG_project_snapshot_template.md](./OG_project_snapshot_template.md)

Raw 버튼 클릭 → 전체 복사 → .md 파일로 저장 → 현재 프로젝트 상태를 채운다.

> ⚠️ 이 링크를 클로드 대화창에 직접 붙여넣지 않는다.
> 클로드는 외부 URL을 가져올 수 없다. 반드시 파일로 저장해서 첨부한다.

### 3. OG 프로젝트에서 첫 대화를 시작한다

`claude.ai` → OG 프로젝트 열기

첨부할 것:
- 채운 스냅샷 템플릿
- openclaw.json (민감 정보 마스킹 후)
- SOUL.md (있다면)
- 관련 로그 (문제가 있는 경우)

```
안녕 OG,
[프로젝트명] 프로젝트야.
스냅샷 채워서 가져왔어. 파일 첨부할게.
지금 겪고 있는 문제: [구체적 설명]
```

## 이런 상황에서 OG에게 가져온다

| 상황 | 예시 |
|------|------|
| openclaw.json 설정이 맞는지 모르겠다 | "이거 검토해줘" |
| Gateway가 안 된다 | "시작이 안 돼. 로그 붙일게" |
| Amy가 Scarlett에게 전달을 안 한다 | "sessions_send가 안 되는 것 같아" |
| 자동화 방법을 모르겠다 | "Cron이랑 Webhook 중 뭐가 맞아?" |
| 버그가 생겼다 | "이런 증상인데 알려진 버그야?" |
| 새 에이전트를 추가하려 한다 | "Natalie 추가하려는데 같이 설계해줘" |
| 파이프라인이 중간에 끊긴다 | "Scarlett → Natalie가 안 이어져" |

## OG가 돌려주는 것

| 산출물 | 내용 |
|--------|------|
| 진단 | 원인 → 판단 → 대안 순서로 정리 |
| 명령어 | echo ✅ 포함, 단계별 실행 가능한 형태 |
| 설정 파일 | 수정된 openclaw.json / SOUL.md — 즉시 사용 가능 |
| 버그 대응 | BUG 번호 + 워크어라운드 + 잔여 리스크 |

---

## 잘 물어보는 법

**이렇게 가져오면 바로 진단한다:**

```
증상 + 로그 + 현재 설정
```

**좋은 예:**
```
Amy가 Scarlett에게 전달했다고 하는데 실제로 안 된 것 같아.
로그는 이래. [로그 붙여넣기]
openclaw.json Amy 설정은 이렇게 돼 있어. [설정 붙여넣기]
```

**나쁜 예:**
```
왜 안 돼?
설정 다 해줘
```

## 발견한 것을 공유하는 방법

OG의 지식은 여러 프로젝트의 경험이 쌓여서 만들어진다.
당신이 발견한 것이 다음 프로젝트를 구한다.

```
✅ OG가 모르던 버그를 발견했을 때
✅ OG가 알려준 방법이 실제로 다르게 동작했을 때
✅ 공식 문서와 실제 동작이 다른 걸 확인했을 때
✅ 특정 환경(macOS 버전, OpenClaw 버전)에서만 나타나는 현상을 발견했을 때
```

공유 형식:
```
환경: [OpenClaw 버전 / OS / 모델]
발견: [무엇을 발견했는가]
재현 방법: [어떻게 하면 재현되는가]
현재 대응: [해결했는가 / 미해결]
```

OG 프로젝트에 가져오면 BUG_REGISTER, LESSONS_LEARNED, RESEARCH_LOG에 반영한다.
